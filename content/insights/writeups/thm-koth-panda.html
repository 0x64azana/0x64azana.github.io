WordPress Administrator Account Discovered via Brute Force

Severity: High

Compliance Gap:
- Weak password enforcement on admin accounts
- No account lockout or rate limiting
- Lack of monitoring for brute force attempts

Policy Violations:
- Weak passwords violate security policies
- No multi-factor authentication (MFA) on admins
- Insufficient access controls

Impact:
An attacker with admin credentials can:
- Upload malicious plugins/themes (web shells)
- Modify PHP files via dashboard editors
- Gain persistent server access
- Exfiltrate data or deface the site

Remediations:

- Implement lockouts or rate limiting
- Enable MFA for admin accounts
- Disable dashboard file editing
- Audit and update plugins/themes regularly
- Monitor login attempts and alerts
